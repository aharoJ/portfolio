# v3 → v4 Changelog (Theme Toggle Only)

## layout.tsx

| Change              | What                                          | Why                                                              |
| ------------------- | --------------------------------------------- | ---------------------------------------------------------------- |
| `color-scheme` meta | `"light"` → `"light dark"` (static)           | Declares support, not active state. More spec-correct.           |
| Blocking script     | Removed `meta[name="color-scheme"]` flip line | CSS drives active scheme now. One less DOM write on first paint. |

## ThemeProvider.tsx

| Change                               | What                                                           | Why                                                                             |
| ------------------------------------ | -------------------------------------------------------------- | ------------------------------------------------------------------------------- |
| **DELETED** `syncColorSchemeMeta()`  | Removed function + 5 call sites                                | Static meta + CSS handles it. Fewer sync points.                                |
| **FIX J** Explicit-choice ref        | `useRef<boolean>` set before toggle, checked by OS handler     | Eliminates micro-race between `applyTheme` and `writeStoredTheme`.              |
| **FIX K** `visibilitychange` cleanup | New `useEffect` removes orphaned `#__theme_no_transitions__`   | Background tabs throttle rAF + setTimeout. Style tag could persist for minutes. |
| Tokenized cleanup                    | Module-scope counter in `withoutTransitions()`                 | Spam-click can't cause older timer to yank style mid-toggle.                    |
| `clearTimeout` on re-toggle          | Cancels previous fallback timer before scheduling new one      | At most ONE pending timeout exists at any time.                                 |
| System pref handler                  | Wrapped in `withoutTransitions()`                              | OS theme change now snaps instantly like manual toggle.                         |
| Cross-tab sync                       | Wrapped in `withoutTransitions()`                              | Other-tab toggles snap instead of animating.                                    |
| bfcache resync                       | Bail-out if `resolved === currentDOM` + `withoutTransitions()` | Prevents unnecessary repaints on back/forward.                                  |
| Meta tag caching                     | `cachedThemeMetas` at module level                             | No repeated `querySelectorAll` on subsequent toggles.                           |
| Touch target                         | `p-2` → `p-3`                                                  | 44×44 min touch target (WCAG 2.5.8).                                            |

## globals.css — NO CHANGES
